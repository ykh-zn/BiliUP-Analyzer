# BiliUP-Analyzer 项目学习指南

## 一、项目全景

### 1.1 项目是什么

一个B站UP主视频数据分析系统，能自动爬取UP主的视频数据，并通过交互式Web看板进行多维度分析。

### 1.2 项目做什么

```
输入UP主UID → 爬取视频数据 → 数据清洗 → 多维分析 → 交互式可视化
```

### 1.3 项目怎么跑

```bash
# 启动爬虫（爬取指定UP主数据）
python run_crawler.py --uid 23920239

# 启动看板（查看分析结果）
python run_dashboard.py
```

## 二、数据层详解（dashboard/data.py）

### 2.1 核心函数

#### `convert_count(value)` — 数值转换

**问题**：B站API返回的数值是字符串，部分带"万"字

**示例**：
```python
convert_count("1.2万")  → 12000
convert_count("333")    → 333
convert_count("0")      → 0
```

**代码逻辑**：
```python
if "万" in value_str:
    num_part = re.sub(r'[^\d.]', '', value_str)  # 去掉"万"字
    return int(float(num_part) * 10000)           # 乘以10000
else:
    return int(float(value_str))                   # 直接转int
```

#### `parse_duration(dur_str)` — 时长转秒数

**问题**：时长格式不统一，有"HH:MM:SS"和"MM:SS"两种

**示例**：
```python
parse_duration("1:30:00")  → 5400秒 (1小时30分)
parse_duration("2:42")     → 162秒  (2分42秒)
parse_duration("45")       → 45秒
```

#### `load_video_data(uid)` — 核心加载函数

**做了4件事**：

```
第1步：JSON → DataFrame
  video_data.json → pd.DataFrame(data)

第2步：类型转换
  播放量/弹幕数/点赞量/投币数/收藏量/转发量/评论数 → int
  发布时间 → datetime
  视频时长 → 时长秒

第3步：计算衍生指标
  弹幕率 = 弹幕数 / 播放量
  投币率 = 投币数 / 播放量
  收藏率 = 收藏量 / 播放量
  转发率 = 转发量 / 播放量
  评论率 = 评论数 / 播放量
  点赞投币比 = 投币数 / 点赞量

第4步：标记有无简介
  有简介 = 1 if 简介非空 else 0
```

#### `clean_data(df)` — 数据清洗

**清洗规则**：
1. 播放量=0 → 排除（无分析价值）
2. 重复标题 → 保留播放量最高的
3. 空标签 → 记录数量，标签分析时跳过

### 3.3 WBI签名机制（核心难点）

**问题**：B站部分API要求请求参数携带WBI签名，防止恶意爬取

**签名流程**：
```
┌─────────────────────────────────────────────────────────┐
│  步骤1：获取密钥素材                                     │
│  GET https://api.bilibili.com/x/web-interface/nav       │
│  → img_url, sub_url                                     │
│  → 提取文件名作为 img_key, sub_key                       │
├─────────────────────────────────────────────────────────┤
│  步骤2：生成mixin_key                                    │
│  拼接 img_key + sub_key                                  │
│  按 MIXIN_KEY_ENC_TAB 索引表重排字符顺序                  │
│  取前32位作为 mixin_key                                   │
├─────────────────────────────────────────────────────────┤
│  步骤3：签名参数                                         │
│  1. 添加 wts = 当前时间戳                                │
│  2. 参数按键名排序                                       │
│  3. 过滤特殊字符 !'()*                                   │
│  4. URL编码                                              │
│  5. 拼接 mixin_key                                       │
│  6. 计算 MD5 哈希 → w_rid                                │
└─────────────────────────────────────────────────────────┘
```

**核心代码**：
```python
MIXIN_KEY_ENC_TAB = [
    46, 47, 18, 2, 53, 8, 23, 32, 15, 50, 10, 31, 58, 3, 45, 35,
    27, 43, 5, 49, 33, 9, 42, 19, 29, 28, 14, 39, 12, 38, 41, 13,
    37, 48, 7, 16, 24, 55, 40, 61, 26, 17, 0, 1, 60, 51, 30, 4,
    22, 25, 54, 21, 56, 59, 6, 63, 57, 62, 11, 36, 20, 34, 44, 52,
]

def get_mixin_key(orig: str) -> str:
    """按索引表重排字符串，取前32位"""
    return ''.join([orig[i] for i in MIXIN_KEY_ENC_TAB])[:32]

def enc_wbi(params: dict, img_key: str, sub_key: str) -> dict:
    """WBI签名完整流程"""
    mixin_key = get_mixin_key(img_key + sub_key)
    params['wts'] = round(time.time())          # 添加时间戳
    params = dict(sorted(params.items()))        # 参数排序
    params = {
        k: ''.join(filter(lambda ch: ch not in "!'()*", str(v)))
        for k, v in params.items()
    }                                            # 过滤特殊字符
    query = urllib.parse.urlencode(params)       # URL编码
    params['w_rid'] = hashlib.md5(
        (query + mixin_key).encode()
    ).hexdigest()                                # MD5签名
    return params
```

### 3.4 数据爬取（requests）

#### API 1：视频列表（需要WBI签名）

**接口**：`GET https://api.bilibili.com/x/space/wbi/arc/search`

**参数**：
- `mid`：UP主UID
- `ps`：每页数量（最大50）
- `pn`：页码
- `order`：排序方式（pubdate=按发布时间）

**返回示例**：
```json
{
    "code": 0,
    "data": {
        "list": {
            "vlist": [
                {"bvid": "BV1xx411c7mD", "title": "视频标题1"},
                {"bvid": "BV1yy411c7mE", "title": "视频标题2"}
            ]
        },
        "page": {
            "count": 778,  // 总视频数
            "pn": 1,       // 当前页码
            "ps": 50       // 每页数量
        }
    }
}
```

**代码逐行解析**：
```python
def fetch_video_list(uid, img_key, sub_key):
    """分页获取所有视频 bvid 列表"""
    video_list = []
    pn = 1
    ps = 50

    while True:
        # 第1步：WBI签名
        params = enc_wbi({
            'mid': uid,
            'ps': ps,
            'pn': pn,
            'order': 'pubdate',
            'tid': 0,
        }, img_key, sub_key)

        # 第2步：发送请求
        resp = session.get(
            'https://api.bilibili.com/x/space/wbi/arc/search',
            params=params
        ).json()

        # 第3步：检查返回码
        if resp['code'] != 0:
            print(f"获取视频列表失败: {resp['message']}")
            break

        # 第4步：解析视频列表
        vlist = resp['data']['list']['vlist']
        total = resp['data']['page']['count']

        if not vlist:
            break

        for v in vlist:
            video_list.append({
                'bvid': v['bvid'],
                'title': v['title'],
            })

        # 第5步：判断是否还有下一页
        print(f"  第 {pn} 页，已获取 {len(video_list)}/{total} 个视频")
        if len(video_list) >= total:
            break

        pn += 1
        time.sleep(random.uniform(0.05, 0.1))  # 请求间隔

    return video_list
```

**关键点**：
- 分页获取，每页最多50个视频
- `while True`循环，直到获取所有视频
- 每次请求间隔0.05-0.1秒，避免触发风控
- 返回所有视频的bvid和标题

### 3.5 请求重试机制

**问题**：B站有风控机制，频繁请求会返回-412错误码

**解决方案**：
```python
def api_get(url, params=None, max_retries=3):
    for attempt in range(max_retries):
        resp = session.get(url, params=params).json()
        code = resp.get('code', -1)
        if code == 0:
            return resp  # 成功
        if code in (-412, -352):
            wait = (attempt + 1) * 5 + random.uniform(1, 3)
            time.sleep(wait)  # 递增等待时间
            continue
        return resp  # 其他错误直接返回
    return resp
```

**请求间隔**：`random.uniform(0.05, 0.1)` 秒，模拟人类操作

### 4.3 关键算法详解

#### 算法1：二八定律分析（analyze_pareto）

**原理**：帕累托法则，即20%的内容贡献80%的收益

**代码逐行解析**：
```python
def analyze_pareto(df):
    # 第1步：计算前20%的视频数量（至少1个）
    n_top = max(1, int(len(df) * 0.2))
    # 例如：778个视频 → 778 × 0.2 = 155个

    # 第2步：获取播放量最高的n_top个视频的播放量总和
    top_views = df.nlargest(n_top, '播放量')['播放量'].sum()
    # 例如：155个视频的播放量总和 = 70,000,000

    # 第3步：计算总播放量
    total = df['播放量'].sum()
    # 例如：全部778个视频的播放量总和 = 100,000,000

    # 第4步：计算占比
    pct = top_views / total * 100
    # 例如：70,000,000 / 100,000,000 × 100 = 70%

    # 第5步：判断头部效应强度
    if pct > 80:
        return f"头部效应极强：{n_top} 个视频（{n_top/len(df)*100:.0f}%）占据了 {pct:.1f}% 的播放量"
    elif pct > 60:
        return f"头部集中度较高：Top {n_top} 视频贡献了 {pct:.1f}% 的播放量"
    else:
        return f"播放量分布较均匀：Top {n_top} 视频仅占 {pct:.1f}%，长尾内容也有不错表现"
```

**输出示例**：`"头部集中度较高：Top 155 视频贡献了 69.7% 的播放量"`

#### 算法2：时长区间分析（analyze_duration_scatter）

**原理**：将视频按时长分组，找出平均播放量最高的区间

**代码逐行解析**：
```python
def analyze_duration_scatter(df):
    # 第1步：过滤掉时长为0的视频
    df_dur = df[df['时长秒'] > 0].copy()

    # 第2步：时长转换为分钟
    df_dur['分钟'] = df_dur['时长秒'] / 60

    # 第3步：定义7个时长区间
    bins = [(0, 1), (1, 3), (3, 5), (5, 10), (10, 20), (20, 60), (60, 999)]

    # 第4步：遍历每个区间，找出平均播放量最高的
    best_avg, best_range = 0, ""
    for lo, hi in bins:
        mask = (df_dur['分钟'] >= lo) & (df_dur['分钟'] < hi)
        if mask.sum() >= 2:  # 至少2个视频才有统计意义
            avg = df_dur[mask]['播放量'].mean()
            if avg > best_avg:
                best_avg = avg
                best_range = f"{lo}-{hi}分钟"

    return f"平均播放量最高的时长区间：{best_range}（均值 {best_avg:,.0f}）"
```

**输出示例**：`"平均播放量最高的时长区间：3-8分钟（均值 126,298）"`

#### 算法3：Pearson相关系数（analyze_views_vs_coinrate）

**原理**：衡量两个变量之间的线性相关程度

**公式**：$r = \frac{\sum(x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum(x_i - \bar{x})^2 \sum(y_i - \bar{y})^2}}$

**判断标准**：
- |r| > 0.3：相关性较强
- |r| ≤ 0.3：相关性较弱
- r > 0：正相关（一个增加，另一个也增加）
- r < 0：负相关（一个增加，另一个减少）

**代码逐行解析**：
```python
def analyze_views_vs_coinrate(df):
    # 第1步：计算Pearson相关系数
    r = df['播放量'].corr(df['投币率'])
    # 例如：r = 0.25

    # 第2步：判断相关性强度
    if abs(r) > 0.3:
        strength = "较强"
        direction = "正" if r > 0 else "负"
    else:
        strength = "较弱"
        direction = ""

    return f"播放量与投币率的Pearson相关系数为{r:.3f}，呈{strength}{direction}相关。"
```

**输出示例**：`"播放量与投币率的Pearson相关系数为0.250，呈较弱正相关。"`

**解读**：播放量高的视频，投币率不一定高。投币行为更多取决于内容质量和情感深度，而非单纯的流量大小。

#### 算法4：标签共现分析（analyze_tag_cooccurrence）

**原理**：统计哪些标签经常一起出现，构建标签组合策略

**代码逐行解析**：
```python
def analyze_tag_cooccurrence(df):
    # 第1步：提取所有标签对
    pair_counts = Counter()
    for tags_str in df['标签'].dropna():
        tags = [t.strip() for t in tags_str.split(',') if t.strip()]
        # 例如：['小米', '小米汽车', '纽北']
        for t1, t2 in combinations(sorted(tags), 2):
            pair_counts[(t1, t2)] += 1
        # combinations(['小米', '小米汽车', '纽北'], 2)
        # → ('小米', '小米汽车'), ('小米', '纽北'), ('小米汽车', '纽北')

    # 第2步：取共现次数最多的前10对
    top_pairs = pair_counts.most_common(10)

    # 第3步：生成分析文本
    if top_pairs:
        examples = "、".join([f'"{p[0]}+{p[1]}"({c}次)' for p, c in top_pairs[:3]])
        return f"共现次数最多的标签组合：{examples}"
    return "标签共现数据不足"
```

**输出示例**：`"共现次数最多的标签组合："小米+小米汽车"(156次)、"小米+雷军"(98次)、"小米汽车+纽北"(45次)"`

#### 算法5：爆款视频识别（analyze_views_wave）

**原理**：用统计方法识别播放量异常高的视频

**代码逐行解析**：
```python
def analyze_views_wave(df):
    # 第1步：计算均值和标准差
    mean = df['播放量'].mean()
    std = df['播放量'].std()

    # 第2步：定义爆款阈值 = 均值 + 2倍标准差
    threshold = mean + 2 * std
    # 例如：mean = 126,298, std = 300,000
    # threshold = 126,298 + 2 × 300,000 = 726,298

    # 第3步：筛选爆款视频
    hot = df[df['播放量'] > threshold]

    # 第4步：生成分析文本
    if len(hot) > 0:
        examples = "、".join(hot.nlargest(3, '播放量')['标题'].tolist())
        return f"发现 {len(hot)} 个爆款视频（播放量 > {threshold:,.0f}），代表：{examples}"
    return "未发现统计意义上的爆款视频"
```

**输出示例**：`"发现 5 个爆款视频（播放量 > 726,298），代表：科幻导演VS科学家、小米SU7 Ultra、撒贝宁聊出宇宙级真相"`

#### 图表5：标签共现网络图（plot_tag_cooccurrence，最复杂）

**代码逐行解析**：
```python
def plot_tag_cooccurrence(df):
    # 第1步：统计标签对的共现次数
    pair_counts = Counter()
    for tags_str in df['标签'].dropna():
        tags = [t.strip() for t in tags_str.split(',') if t.strip()]
        for t1, t2 in combinations(sorted(tags), 2):
            pair_counts[(t1, t2)] += 1

    # 第2步：取共现次数最多的前15个标签
    top_tags = set()
    for (t1, t2), count in pair_counts.most_common(50):
        top_tags.add(t1)
        top_tags.add(t2)
        if len(top_tags) >= 15:
            break
    top_tags = list(top_tags)[:15]

    # 第3步：圆形布局计算节点坐标
    n = len(top_tags)
    node_x, node_y, node_text, node_size = [], [], [], []
    for i, tag in enumerate(top_tags):
        angle = 2 * np.pi * i / n               # 均匀分布在一个圆上
        node_x.append(np.cos(angle))             # x坐标
        node_y.append(np.sin(angle))             # y坐标
        node_text.append(tag)
        # 节点大小 = 该标签出现次数
        count = sum(1 for tags_str in df['标签'].dropna() if tag in tags_str)
        node_size.append(max(10, count * 2))

    # 第4步：计算边的坐标（共现关系）
    edge_x, edge_y = [], []
    for (t1, t2), count in pair_counts.items():
        if t1 in top_tags and t2 in top_tags:
            i1, i2 = top_tags.index(t1), top_tags.index(t2)
            # 画线连接两个节点
            edge_x.extend([node_x[i1], node_x[i2], None])  # None表示断开
            edge_y.extend([node_y[i1], node_y[i2], None])

    # 第5步：创建图形
    fig = go.Figure()

    # 添加边（连线）
    fig.add_trace(go.Scatter(
        x=edge_x, y=edge_y,
        mode='lines',
        line=dict(width=0.5, color='#888'),
        hoverinfo='none'
    ))

    # 添加节点（标签）
    fig.add_trace(go.Scatter(
        x=node_x, y=node_y,
        mode='markers+text',
        marker=dict(size=node_size, color='#00a1d6'),
        text=node_text,
        textposition='top center',
        hovertemplate='%{text}<extra></extra>'
    ))

    # 第6步：配置布局
    fig.update_layout(
        title='标签共现网络',
        title_font=TITLE_FONT,
        showlegend=False,
        height=500,
        xaxis=dict(visible=False),
        yaxis=dict(visible=False)
    )
    return fig
```

**关键点**：
- `combinations(sorted(tags), 2)`：生成所有标签对（不重复）
- 圆形布局：节点均匀分布在一个圆上
- `edge_x.extend([..., None])`：None表示断开，避免节点间连线
- 节点大小映射标签出现次数

### 6.4 看板内爬虫集成（技术难点）

**问题**：Streamlit每次交互都重新执行脚本，无法直接在主线程运行长时间任务

**解决方案**：subprocess + threading + Queue

```
用户点击"开始爬取"
        │
        ▼
subprocess.Popen() 启动爬虫子进程
        │
        ▼
threading.Thread() 后台线程读取stdout
        │
        ▼
queue.Queue() 线程间传递日志
        │
        ▼
session_state 保存进程引用和队列
        │
        ▼
st.rerun() 每0.5秒刷新页面，显示最新日志
```

**关键代码**：
```python
# 启动爬虫
process = subprocess.Popen(
    ["python", "run_crawler.py", "--uid", uid],
    stdout=subprocess.PIPE,
    stderr=subprocess.STDOUT,
    env={**os.environ, 'PYTHONIOENCODING': 'utf-8'},  # 解决Windows编码
    bufsize=1
)

# 后台线程读取日志
def read_output(process, log_queue):
    for line in iter(process.stdout.readline, b''):
        log_queue.put(line.decode('utf-8', errors='replace'))

thread = threading.Thread(target=read_output, args=(process, log_queue))
thread.start()

# 停止爬虫
if st.button("停止"):
    process.terminate()
```

## 七、关键技术点总结

### 7.1 WBI签名机制
- B站API的安全验证机制
- 需要img_key + sub_key + 索引表 + MD5哈希
- 只有视频列表API需要签名，视频详情API不需要

### 7.2 Selenium vs requests
- Selenium：仅用于首次登录获取Cookies（重量级）
- requests：用于所有数据爬取（轻量级）

### 7.3 pandas数据处理
- JSON → DataFrame → 类型转换 → 衍生指标 → 清洗
- 核心：`apply()` + 自定义函数实现批量转换

### 7.4 Plotly图表
- 所有图表返回`go.Figure`对象
- Streamlit通过`st.plotly_chart()`渲染
- 支持hover、zoom、pan等交互操作

### 7.5 Streamlit状态管理
- `session_state`保存跨交互的状态
- 每次交互重新执行脚本，需要用session_state保持数据

## 八、常见面试问题

### Q1：为什么用Selenium而不是直接requests登录？
**A**：B站登录需要扫码，无法通过API模拟。Selenium控制浏览器让用户手动扫码，获取Cookies后保存，后续用requests携带Cookies请求。

### Q2：WBI签名的作用是什么？
**A**：B站的安全机制，防止恶意爬取。通过时间戳保证请求时效性，参数排序保证签名一致性，MD5哈希保证不可伪造性。

### Q3：为什么用Streamlit而不是Flask/Django？
**A**：Streamlit专为数据应用设计，代码量少（400行实现5个Tab），内置Plotly支持，适合快速构建数据分析看板。

### Q4：如何处理B站的反爬机制？
**A**：
1. 请求间隔0.05-0.1秒，模拟人类操作
2. 遇到-412错误码自动等待重试
3. 递增等待时间（5s, 10s, 15s）
4. 最大重试3次

### Q5：数据量大时如何优化？
**A**：
1. 当前方案：单线程爬取，适合中小规模
2. 可优化：多线程/异步爬取、数据库存储、缓存机制

### Q6：为什么选择JSON而不是数据库存储？
**A**：
1. JSON简单，无需安装数据库
2. 便于前端直接读取
3. 适合单机小规模应用
4. 可扩展：未来可改为SQLite/MySQL

### Q7：Plotly和matplotlib的区别？
**A**：
1. Plotly：交互式图表，支持hover/zoom/pan，适合Web应用
2. matplotlib：静态图表，适合论文/报告
3. 本项目：Plotly用于看板交互，matplotlib用于词云（wordcloud库依赖）
